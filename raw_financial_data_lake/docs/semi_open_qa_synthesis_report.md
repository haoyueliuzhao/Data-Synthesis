# 半开放 QA 合成链路报告

## 一、概述

半开放 QA 是介于封闭式问答与完全开放生成之间的数据合成形态。在本项目中，它具体表现为“证据给定的金融分析”：系统不再要求回答只有一个固定值或唯一措辞，而是允许模型在既定证据范围内组织观点、安排分析顺序、选择分析侧重点，并从多个合法结论中选择最合适的表达。

从数据组织上看，它是与封闭 QA 并行的独立分析链路，而不是普通 QA 的一个模板类型。本报告沿用“半开放 QA”这一广义称呼，强调其问题输入、证据条件、分析回答和质量评价共同构成一类问答数据。

这种开放是有边界的。可以开放的是语言、结构和强调重点；不能开放的是事实、数字、实体、期间、证据归属、因果关系以及投资结论。其目标不是让模型自由发挥，而是在可验证事实之上生成更接近真实分析工作的回答。

## 二、主要作用

### 1. 衔接结构化事实与自然分析

标准化事实适合检索、计算和核验，但不能直接形成高质量分析语料。半开放链路把收入、利润、现金流、利润率、杠杆和同行位置等事实转化为可解释的趋势、比较、风险信号和综合判断，使底层数据能够服务于分析型模型训练与评测。

### 2. 提供比封闭 QA 更真实的任务形态

封闭 QA 主要验证“答案是否正确”，适合查值、计算、排序和确定性推理。真实金融分析还要求模型识别多条证据之间的关系、处理正反信号、给出限制条件，并在多个合理结论中作出选择。半开放 QA 正是为了覆盖这一能力层级。

### 3. 形成可控的分析训练数据

完全自由的分析文本难以判断哪些内容由证据支持，也容易混入无依据因果、预测和投资建议。半开放链路先限定证据、主张和结论空间，再允许模型进行受控表达，因此更适合生成：

- 证据给定的分析能力评测集；
- 强调事实约束和风险意识的监督微调数据；
- 用于训练推理、证据对齐和结论选择的过程数据；
- 用于检验模型是否忽略反向证据的对抗样本。

### 4. 将“自然度”与“正确性”分开治理

LLM 负责表达组织，但不拥有事实裁决权。最终样本是否可用，由独立验证环节根据原始证据重新判断。这种分工可以在增加语言多样性的同时，避免把模型自我声明的正确性当成事实。

## 三、链路核心

整体链路可以概括为：

标准化事实与知识图谱
→ 可复算金融信号
→ 固定证据包
→ 最小可验证主张
→ 合法结论集合
→ 受控 LLM 表达
→ 独立验证
→ 防泄漏切分与数据导出

### 1. 证据先于文本

系统先确定本次分析可以使用哪些公司、指标、期间和来源，再生成文本。任何分析都必须绑定到明确证据，不能先写结论再寻找支持材料。

### 2. 从事实抽象为金融信号

单个事实通常不足以形成分析价值。链路会把事实组合成趋势、增长、利润与现金流背离、利润率变化、资产效率和同行百分位等信号。信号是事实与自然语言主张之间的中间层，使分析既可解释，也可重新计算。

### 3. 将回答拆成可验证主张

回答不是一个不可分割的长文本，而是由若干最小主张组成。每个主张都明确由哪些信号支持、是否存在反向证据、是否必须出现，以及与其他主张是依赖还是冲突关系。

### 4. 结论不是无限生成

同一组证据可能支持多个合理侧重点，但并不支持任意结论。系统会根据当前证据状态形成合法结论集合，模型只能从中选择。这样既保留半开放特征，也避免模型从“增长放缓”跨越到“公司管理失效”或“应当卖出”等无依据判断。

### 5. 反向证据是必要组成部分

半开放分析不能只挑选支持主结论的证据。当利润增长但现金流恶化、收入提升但利润率下降，或者同行排名在不同指标上出现冲突时，回答必须体现这种制约关系。遗漏关键反向证据会导致样本不合格。

### 6. LLM 负责组织，不负责创造事实

模型可以调整主张顺序、选择合法结论和注册表达方式，但不能新增数字、实体、期间、指标、原因、预测或建议。核心句子的金融含义仍由受控结构决定。

### 7. 独立验证形成最后防线

生成完成后，验证器会重新计算信号、检查证据覆盖、核对主张与结论、确认反向证据是否被处理，并检查文本中的数字和单位。API 调用成功只代表模型返回了内容，不代表样本已经合格。

## 四、当前能力范围

当前链路主要覆盖三类任务：

### 1. 经营趋势总结

围绕收入、净利润和经营现金流的多期变化，识别增长、下降、波动和趋势一致性，形成对经营表现的描述性总结。

### 2. 增长质量诊断

综合增长、利润与现金流背离、利润率和资产效率，判断增长是否得到盈利质量和现金质量支持，并在必要时加入风险限制。

### 3. 同行位置分析

在完整且口径一致的同行范围内，比较收入增长、净利率和杠杆水平，描述公司相对位置及其优势与约束。

这些能力覆盖了“描述、比较、诊断、风险总结”四种基础分析动作，但尚未进入预测、估值和投资决策。

当前已完成一轮覆盖三类模式的 6 条真实 API 小规模验证，全部通过独立校验且没有最终回退。这证明链路在小样本上具备可行性，但不能替代新语义契约下的全量规模验收。

## 五、核心价值判断

### 1. 半开放不等于弱约束

链路的价值来自“有限开放”。如果完全锁死文本，数据会退化为模板问答；如果完全开放结论，又会失去可验证性。当前设计把开放空间集中在表达与组织层，把事实和结论边界保留在确定性系统中。

### 2. 多样性应建立在语义等价之上

有价值的多样性包括不同分析顺序、不同证据组合、不同正反信号结构和不同合法结论，而不仅是同一句话的同义改写。后续评估应优先关注分析结构多样性，而不是单纯统计词汇差异。

### 3. 反向证据决定分析质量上限

只会总结正向事实的数据，很容易训练出过度自信的模型。是否能够发现、保留并正确表达反向证据，是半开放分析区别于普通摘要任务的核心指标。

### 4. 可验证性比生成规模更重要

半开放样本的单条成本高于封闭 QA，但质量风险也更大。扩大规模前，应优先保证信号可复算、同行范围完整、主张可对齐、结论合法以及切分无泄漏。

## 六、当前局限性

### 1. 分析主题仍然有限

目前只有经营趋势、增长质量和同行位置三类主模式。对资本结构、营运资金、分部表现、盈利持续性、宏观敏感度和事件影响的覆盖不足。

### 2. 证据以结构化财务事实为主

当前链路尚未把公告正文、财报段落、管理层讨论、表格脚注和事件材料作为正式证据。因此它能判断“发生了什么”，但难以可靠回答“公司如何解释”或“事件为何发生”。

### 3. 不支持因果、预测与投资建议

收入和现金流背离只能支持风险提示，不能直接推导管理问题、舞弊、未来业绩或买卖建议。这是当前能力边界，也是必要的安全边界。

### 4. 同行分析受数据完整性制约

同行结论依赖完整且可比的公司集合。行业分类缺失、指标口径不一致、币种和期间不一致，都会明显减少可生成样本，甚至使某个同行范围无法使用。

### 5. 表达多样性仍有上限

为了保证语义可验证，当前核心表达来自受控形式。这样能提高可靠性，但会限制语言风格、长文本组织和复杂论证的自然程度。需要避免数据长期呈现明显模板化。

### 6. 成本与时延较高

半开放生成涉及多信号组织、主张选择、反向证据处理和独立验证，单条样本的模型调用成本和推理时间高于封闭 QA。规模扩大后需要重点关注失败重试、模型选择和边际数据价值。

### 7. 全量新契约验证仍需补齐

历史 150 条真实 API 运行实现了全部 HTTP 和结构化响应成功，但旧版语义校验发现两条风险主张方向错误。这说明接口成功率不能代表分析正确率。新的语义框架已替代旧规则，小规模真实运行已通过，但仍需要用新契约完成一次完整规模的发布前验证。

## 七、后续拓展方向

### 近期方向

1. 完成新语义契约下的全量验证，建立稳定的通过率、回退率、成本和多样性基线。
2. 扩展资本结构、现金转换、利润持续性、营运资金和多指标质量诊断模式。
3. 增加冲突证据、边界案例和困难负例，强化模型对风险限制与不确定性的表达。
4. 引入人工抽检维度，评估自然度、分析价值、信息密度和是否存在“正确但无用”的回答。
5. 扩展中英文表达，并分别评估语言自然度与金融术语一致性。

### 中期方向

1. 接入文档级证据，使分析能够引用财报正文、公告段落和表格脚注。
2. 增加事件驱动分析，将财务变化与可验证事件按时间关系连接，但仍不自动声明因果。
3. 引入跨来源证据协调，处理 SEC、宏观数据、公司公告和第三方数据之间的口径差异与冲突。
4. 构建检索增强任务，让模型先从受控证据池中选择相关证据，再完成主张和结论组织。
5. 建立偏好数据与验证器负例，训练模型识别“语言自然但证据不足”的答案。

### 长期方向

1. 从单公司和静态同行分析扩展到跨周期、跨行业和宏观联动分析。
2. 建立不确定性分级，使回答能够区分事实确认、弱信号、冲突证据和证据不足。
3. 支持更长的多段分析，同时保持每个段落和结论都可追溯到证据与主张。
4. 探索受约束的情景分析。情景参数必须由任务明确给定，输出只能描述条件结果，不能伪装成预测。
5. 形成覆盖封闭 QA、半开放分析、证据检索和验证器评测的统一金融推理数据体系。

## 八、建议的评估框架

后续不应只以样本数量衡量建设进度，建议持续关注以下维度：

- 事实正确性：数字、单位、实体和期间是否准确；
- 证据完整性：支持与反向证据是否覆盖充分；
- 主张忠实度：文本是否准确表达既定信号；
- 结论合法性：结论是否处于当前证据允许范围；
- 分析多样性：模式、信号组合、冲突结构和表达是否丰富；
- 数据泄漏：同一实体、同行范围或证据窗口是否跨集合泄漏；
- 自然度与有用性：回答是否像真实分析，而不是字段拼接；
- 运行效率：单条成本、时延、重试和最终合格率是否可接受。

## 九、结论

半开放 QA 合成链路的核心意义，是把“可验证事实”提升为“可验证分析”。它比封闭 QA 更接近真实金融任务，又比自由文本生成更容易控制事实风险。

当前方案已经建立了证据、信号、主张、结论和独立验证之间的基本闭环，适合用于训练和评估证据约束下的分析能力。下一阶段的重点不应只是扩大样本量，而应优先扩展分析主题和证据类型、提高结构多样性、完成新契约下的全量验证，并逐步引入文档证据、检索和不确定性表达。
